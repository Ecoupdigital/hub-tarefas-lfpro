## Achados de Pesquisa (Inline)

- **Pacotes obrigatorios** (verificados em blocknotejs.org/docs):
  - `@blocknote/core` — core do editor (TipTap por baixo)
  - `@blocknote/react` — hooks (`useCreateBlockNote`) e componentes React
  - `@blocknote/mantine` — UI components prontos (`BlockNoteView`) e `style.css`
- **CSS imports obrigatorios** (no ponto de uso):
  - `@blocknote/core/fonts/inter.css` — fonts default (vamos sobrescrever)
  - `@blocknote/mantine/style.css` — estilos da UI
- **API basica:**
  ```tsx
  import { useCreateBlockNote } from "@blocknote/react";
  import { BlockNoteView } from "@blocknote/mantine";
  const editor = useCreateBlockNote({ initialContent });
  return <BlockNoteView editor={editor} theme={theme} onChange={...} />;
  ```
- **Componente e uncontrolled.** `initialContent` so e lido na criacao do hook. Para atualizar externamente: `editor.replaceBlocks(...)`.
- **Tema:** `BlockNoteView` aceita prop `theme` (objeto Theme do BlockNote com `colors`, `borderRadius`, `fontFamily`) ou prop `theme="light" | "dark"`. Tokens via CSS variables podem complementar.
- **Compatibilidade:** BlockNote roda em cima de @tiptap/core. Projeto ja tem @tiptap/react 3.20 instalado (`package.json:50-55`). Verificar que `@blocknote/core` aceita peer dep tiptap 3.
- **Vite:** `@blocknote/mantine/style.css` precisa ser importado no entry para nao quebrar HMR. Convencao: importar UMA vez em `src/index.css` via `@import` ou no ponto de uso (usar no ponto de uso para code-split por lazy load).

```tsx
import React, { useEffect, useMemo } from 'react';
import { useCreateBlockNote } from '@blocknote/react';
import { BlockNoteView } from '@blocknote/mantine';
import type { Block, PartialBlock } from '@blocknote/core';
import { useTheme } from 'next-themes';

import { lfproBlockNoteLightTheme, lfproBlockNoteDarkTheme } from './blocknote-theme';

// Imports de CSS obrigatorios do BlockNote.
// Nao remover. Os overrides finos vivem em src/styles/blocknote-overrides.css.
import '@blocknote/core/fonts/inter.css';
import '@blocknote/mantine/style.css';

export interface PageEditorProps {
  /** Conteudo inicial em formato BlockNote. Lido apenas na montagem. */
  initialContent?: PartialBlock[];
  /** Callback disparado a cada mudanca (debounce e responsabilidade do consumidor). */
  onChange?: (blocks: Block[]) => void;
  /** Bloqueia edicao (leitura apenas). */
  editable?: boolean;
  /** className do container externo. */
  className?: string;
}

/**
 * PageEditor — editor base BlockNote tematizado para LFPro.
 *
 * Componente uncontrolled (segue API do BlockNote): `initialContent` so e lido na
 * primeira renderizacao. Para atualizar externamente use `editor.replaceBlocks`.
 *
 * Tema acompanha o tema global (light/dark) via next-themes.
 *
 * Conexao com plano 01-04: hook de auto-save chama `onChange` com debounce.
 * Conexao com plano 01-05: extensoes customizadas (mention/embed) entram aqui.
 */
const PageEditor: React.FC<PageEditorProps> = ({
  initialContent,
  onChange,
  editable = true,
  className,
}) => {
  const { resolvedTheme } = useTheme();

  const editor = useCreateBlockNote({
    initialContent: initialContent && initialContent.length > 0 ? initialContent : undefined,
  });

  const theme = useMemo(
    () => (resolvedTheme === 'dark' ? lfproBlockNoteDarkTheme : lfproBlockNoteLightTheme),
    [resolvedTheme]
  );

  // Atualiza editable dinamicamente.
  useEffect(() => {
    editor.isEditable = editable;
  }, [editor, editable]);

  return (
    <div className={className ?? 'w-full max-w-3xl mx-auto px-4'}>
      <BlockNoteView
        editor={editor}
        theme={theme}
        editable={editable}
        onChange={() => {
          if (onChange) {
            onChange(editor.document);
          }
        }}
      />
    </div>
  );
};

export default PageEditor;
```

Notas:
- NAO usar em-dash em comentarios.
- Sem `any`. Tipos vem do `@blocknote/core`.
- `useTheme` ja e usado pelo projeto (`next-themes` 0.3.0 esta em package.json).
- A propriedade `editor.document` retorna `Block[]` atualizado em qualquer momento.

# Fase 02 Plano 02-06: Views Tabela/Kanban/Calendar com mode='database'

**Objetivo:** Reusar componentes `BoardTable`, `BoardKanban`, `BoardCalendar` dentro do bloco database via prop `mode='database'`. Isolar contexto (database tem seu proprio boardId, nao deve interferir no board ativo do app). Aplicar subset de 8 tipos em CreateColumnModal quando criando coluna em database.

## Pesquisa inline aplicada

- **BoardTable/Kanban/Calendar leem activeBoardId via useApp().** Para reuso em database, precisamos: (a) prop `boardId` opcional que override o context, OU (b) provider local que sobrescreve AppContext. CONTEXT.md sugere alternativa (b). Faremos via novo `DatabaseBoardContext` que envolve um sub-AppContext minimo.
- **Alternativa mais simples:** Aceitar prop `boardId` no Board* e passar adiante para todos os hooks internos (useItems, useColumns etc.). Trade-off: muitos toques em arquivos grandes. Optamos por (b) - Provider local - pois minimiza modificacoes em BoardTable/Kanban/Calendar.
- **AppContext atual:** `BoardContext.tsx` carrega dados via hooks com base em `activeBoardId` do UIContext. Pra Database, criamos um wrapper que monta esses providers com um boardId injetado em vez do UIContext.activeBoardId.

## Contexto

@src/context/AppContext.tsx - agregador de providers
@src/context/UIContext.tsx - activeBoardId, selectedItem
@src/context/BoardContext.tsx - carrega boards/groups/columns/items via hooks
@src/components/board/BoardTable.tsx - usa useApp() pra activeBoardId, columnValues
@src/components/board/BoardKanban.tsx, BoardCalendar.tsx - idem
@src/components/modals/CreateColumnModal.tsx - UI de criar coluna (sera filtrada)
@src/types/database.ts (02-01) - DATABASE_COLUMN_TYPES (8 tipos)

import { BoardProvider } from '@/context/BoardContext';

interface DatabaseBoardContextProps {
  boardId: string;
  children: React.ReactNode;

/**
 * Wrapper que monta um BoardProvider local com boardId fixo,
 * permitindo BoardTable/Kanban/Calendar funcionarem dentro do bloco database
 * SEM interferir no activeBoardId do app principal.
 *
 * Estrategia:
 *  - UIContext do app principal continua com activeBoardId do board "real" ativo (ou null em /page/:id)
 *  - DatabaseBoardContext sobrescreve activeBoardId localmente para o sub-arvore renderizar dados deste board
 *  - FilterContext + SelectionContext sao re-instanciados localmente pra filtros/selecao nao conflitarem
 */
const DatabaseBoardContext: React.FC<DatabaseBoardContextProps> = ({ boardId, children }) => {
  // BoardProvider espera AppContext de UIContext com activeBoardId; reaproveitamos
  // o pattern com um proxy minimo.
  // Implementacao: ver alternativas:
  //  (a) UIContext aceita prop initialBoardId; criar instancia nova local
  //  (b) Modificar BoardProvider pra aceitar boardId via prop opcional override
  // OPCAO (b) e MENOS invasiva.
  return (
    <FilterProvider>
      <SelectionProvider>
        <BoardProvider boardIdOverride={boardId}>{children}</BoardProvider>
      </SelectionProvider>
    </FilterProvider>
  );
};

export default DatabaseBoardContext;

import React, { useEffect } from 'react';
import { useBoardViews } from '@/hooks/useBoardViews';
import { useBoards } from '@/hooks/useSupabaseData';
import DatabaseBoardContext from './DatabaseBoardContext';
import BoardTable from '@/components/board/BoardTable';
import BoardKanban from '@/components/board/BoardKanban';
import BoardCalendar from '@/components/board/BoardCalendar';
import DatabaseListView from './DatabaseListView'; // criado em 02-07
import type { DatabaseViewType } from '@/types/database';

interface Props {
  boardId: string;
  /** Id da view ativa (vem de DatabaseViewTabs). Se null, usa is_default ou primeira. */
  activeViewId?: string | null;
}

const DatabaseViewRenderer: React.FC<Props> = ({ boardId, activeViewId }) => {
  const { data: views = [] } = useBoardViews(boardId);

  // Determina view ativa
  const activeView = activeViewId
    ? views.find((v) => v.id === activeViewId)
    : views.find((v) => v.is_default) ?? views[0];

  if (!activeView) {
    return (
      <div className="px-3 py-4 text-sm text-muted-foreground">
        Carregando views...
      </div>
    );
  }

  const viewType = activeView.view_type as DatabaseViewType;

  return (
    <DatabaseBoardContext boardId={boardId}>
      {viewType === 'table' && <BoardTable mode="database" />}
      {viewType === 'kanban' && <BoardKanban mode="database" />}
      {viewType === 'calendar' && <BoardCalendar mode="database" />}
      {viewType === 'list_detailed' && <DatabaseListView mode="database" />}
    </DatabaseBoardContext>
  );
};

export default DatabaseViewRenderer;
